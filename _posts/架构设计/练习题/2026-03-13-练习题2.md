# 问题8  
## Redis扣减库存成功，但MQ发送失败会不会库存丢失？

### 我的回答
1. Redis扣减库存成功后，DB未扣减库存，实际库存未减少，但缓存库存减少  
2. 原因：MQ消息发送失败，订单服务未消费消息，DB未扣库存  
3. 解决：发送MQ时同时保存DB可靠消息，通过定时任务扫描重试发送  

缺点：依赖可靠消息定时任务扫描

### 更优回答
会出现库存丢失问题：

Redis扣库存成功 → MQ发送失败 → 订单服务未消费 → 未创建订单  

解决方案：**本地消息表（可靠消息）**

流程：

1. 发送MQ前写入消息表
2. 发送MQ
3. 若发送失败，通过定时任务重试
4. 保证MQ消息最终投递成功

### Redis与DB一致性

无法保证强一致，只能保证：

**最终一致性**

# 问题9  
## MQ消费成功，但DB写订单失败怎么办？

### 我的回答
1. 消费端支持重试：DB失败回滚事务，返回失败让MQ重新投递  
2. 幂等处理：秒杀活动ID + 用户ID判断是否已创建订单  

### 更优回答

1. **MQ重试机制**  
   - DB失败 → 回滚事务 → 返回失败 → MQ重新投递

2. **幂等处理**  
   - activityId + userId 查询是否已下单

3. **数据库唯一键兜底**  
   - activityId + userId 建唯一索引  
   - insert重复时捕获异常

4. **补偿处理**  
   - 超过最大重试次数 → 死信队列  
   - 告警 + 人工处理

# 问题10  
## Redis扣库存成功，但用户取消订单或支付失败，库存如何恢复？

### 我的回答

1. 记录库存恢复记录  
   - 取消订单时新增恢复记录（状态=处理中）  
   - 发送恢复库存MQ消息

2. 恢复库存流程  

消费端：

- 查询恢复记录状态（幂等）
- 本地事务：DB库存+1 + 更新恢复记录状态
- Redis Lua incr库存

1. **记录库存恢复日志**

取消订单：
